# AI Development Rules

Any AI assistant contributing to this repository must follow these rules:

1. Never produce notebook-style code.

2. Keep functions small and focused.

3. Every public function must have:
   - Type hints
   - Docstrings

4. Prefer pathlib over os.path.

5. Never hardcode dataset paths.

6. Always write reusable modules.

7. Every new feature must include a test.

8. Follow Single Responsibility Principle.

9. Prefer readability over clever code.

10. Keep commits feature-based.

11. Maintain modular architecture.

12. Explain major architectural decisions before implementation.

13. Never duplicate functionality.

14. Keep the project generic enough to support any football competition.

15. Preserve the project's vision of becoming a reusable football analytics engine, not just a World Cup predictor.
